# Credit Card Fraud Detection — SQL + Power BI

## 📊 Dataset
- 568,630 transactions
- 31 columns including Amount and Class
- Binary classification: 0 = Legitimate, 1 = Fraud

## 🔍 Business Questions Solved

### SQL Analysis (MySQL Workbench)
1. Separated fraud vs legitimate transactions
2. Calculated fraud statistics (count, sum, avg, max, min)
3. Identified top 10 highest fraud transactions
4. Analyzed fraud rate percentage
5. Detected above-average fraud amounts using subqueries
6. Broke down transactions by amount range

### Power BI Dashboard
7. Built KPI cards for total transactions, fraud count, 
   fraud rate and fraud amount
8. Created interactive bar and donut charts
9. Analyzed transaction trends by sequence
10. Built dynamic slicer for real-time filtering

## 🛠️ Tools Used
- Python (Pandas, Sklearn) — Data cleaning & ML
- MySQL Workbench — SQL Analysis
- Power BI Desktop — Dashboard

## 📈 Key Insights
- Fraud rate: 50% (balanced dataset)
- Average fraud amount: $12,057
- Total fraud amount: $3.43K
- High value frauds (>$1000): identified and flagged

## 🚀 Results
Successfully analyzed transaction data and identified patterns that can help detect potentially fraudulent activities.
